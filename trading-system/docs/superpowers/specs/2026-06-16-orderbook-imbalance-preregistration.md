# 사전등록: 오더북 top-of-book 임밸런스 → 단기 수익률 예측력 분석

> 작성일: 2026-06-16 · 대상: `order_books` 테이블(캐처 적재분) · **순수 SQL 분석, 코드/전략/엔진 불변**
> 사전등록 문서 — **숫자를 보기 전에** 가설·신호정의·horizon·판정 기준·다중비교 보정을 확정한다.
> 이건 **신호 연구(signal study)**다. 전략이 아니다. 여기서 신호가 살아남아야만 전략 구현으로 넘어간다.

## 0. 배경 — 왜 이 분석인가

가격방향 전략 4계열(순수돌파·추세필터돌파·평균회귀·일봉돌파)이 1m/5m/1h/1d 전부
OOS 무엣지로 falsify됐다(PROGRESS.md 함정 #1~#5). 메타 교훈: **같은 BTC/ETH 2023-26 레짐에서
"가격이 레벨을 넘는다" 변형을 더 돌리는 건 family-wise 거짓양성률만 키운다.** 엣지가 있다면
가격방향(캔들 OHLCV)이 아닌 **다른 차원**에 있다.

오더북 top-of-book 임밸런스는 가격방향과 **직교한** 마이크로구조 신호다 — 캔들이 아니라
**호가창의 매수/매도 압력 비대칭**을 본다. 단, 검증하려면 데이터가 필요해서 캐처를 Hetzner에
배포해 1초 샘플 top-of-book를 몇 주간 적재 중이다(2026-06-16~). 이 문서는 데이터가 충분해지면
**전략을 짜기 전에 먼저 돌릴 신호 연구**를 사전등록한다.

⚠️**핵심 규율**: 가격방향 패밀리가 4번 죽은 이유는 IS 숫자를 믿고 그리드를 돌렸기 때문이다.
임밸런스도 똑같이 **신호 연구(IC/분위수)부터** 하고, 거기서 살아남아야만 전략 구현 + 동일한
워크포워드 + OOS + 수수료 + 적대적 리뷰로 넘어간다. **신호 연구가 죽으면 거기서 STOP** —
"전략으로 짜면 다를지도"라며 진행하지 않는다(그게 family-wise 거짓양성의 시작).

## 1. 가설 (사전등록 — 한 문장)

> **top-of-book 임밸런스 `I = bid_size / (bid_size + ask_size)`가 다음 H초 mid-price 수익률을
> 단조(monotone)·통계유의하게 예측한다** (I 높음=매수압력=다음 수익률 양(+) 방향).

이 부호 가정(I↑ → forward return↑)을 **미리** 못박는다. 데이터를 보고 부호를 뒤집어 해석하면
사전등록이 무의미해진다. 만약 IC가 **유의하게 음(−)** 으로 나오면 그건 "역방향 신호 발견"이 아니라
**가설 부호 위반**이므로 별도 가설로 새 홀드아웃에서 재검증해야 한다(아래 7).

## 2. 신호 정의 (사전등록 — 고정)

- **임밸런스**: `I = bid_size / (bid_size + ask_size)`, 범위 (0,1). 0.5=균형, >0.5=매수우세.
  - `bid_size + ask_size = 0`인 행은 제외(0 나눗셈 방지·무의미 호가).
- **기준 가격(mid)**: `mid = (best_bid + best_ask) / 2`. **체결가/close 아님** — bid-ask bounce로 인한
  가짜 수익률을 피하려고 mid를 쓴다(top-of-book bounce trap, 아래 4).
- **forward return at horizon H**: `r_H(t) = mid(t+H) / mid(t) - 1`.
  - `t+H` 행은 "정확히 H초 뒤" 행이 아니라 **`event_time`이 `[t+H, t+H+tol)` 구간의 첫 행**으로
    매칭(1초 샘플이라 tol=2초). 정확히 t+H 행이 없으면(피드 stall 등) 그 샘플은 버린다 —
    **수익률을 보간(interpolate)하지 않는다**(없는 데이터를 만들지 않음).

## 3. Horizon (사전등록 — 고정)

**H ∈ {10s, 30s, 60s}** 세 개를 동시에 등록한다. top-of-book 임밸런스는 전형적으로 초~분 단위
단기 예측력이라 1초 샘플과 잘 맞는다. 세 horizon을 같이 봐서 **IC term-structure**(예측력이
horizon 따라 어떻게 감쇠하는지)를 본다.

5m/15m 등 장기는 이 사전등록에 **포함하지 않는다** — 노브를 늘리면 다중비교만 키운다. 단기 셋이
전부 죽으면 거기서 STOP. (장기가 보고 싶으면 그건 새 사전등록 + 새 홀드아웃.)

## 4. 적대적 설계리뷰 — 반드시 피해야 할 함정 (이 문서가 존재하는 이유)

가격방향 패밀리에서 배운 대로, **숫자를 보기 전에** 방법론 결함을 못박는다:

1. **bid-ask bounce trap**: forward return을 체결가/close로 재면, 가격이 안 움직여도 bid↔ask 사이를
   튀는 것만으로 가짜 수익률이 생기고 임밸런스와 기계적 상관이 뜬다. → **mid-price로만 측정**(2).
2. **중복 표본(overlapping windows) trap**: 1초마다 forward return을 겹쳐 계산하면 인접 표본이
   강하게 자기상관 → 유효표본수 과대 → IC 유의성 과대평가. → **비중복(non-overlapping) 표본만 사용**:
   각 horizon H에 대해 H초 간격으로 표본 추출(10s는 10초마다, 60s는 60초마다 한 점). IC는 이
   비중복 표본 위에서만 계산. (전체 행으로도 참고치는 내되, **판정은 비중복 IC로만**.)
3. **거래소 풀링(pooling) trap**: binance/bybit/bitget는 1초 샘플이 동기화돼 있지 않고, bybit는
   delta 채널이라 top-of-book가 더 stale할 수 있다. 셋을 합치면 같은 시장 움직임을 3중계산 +
   거래소별 마이크로구조 차이를 뭉갠다. → **거래소×심볼별로 따로 IC 계산**(6피드 각각 독립 판정).
   풀링은 절대 안 함. 한 거래소에서 사는데 다른 거래소에서 죽으면 그건 그 거래소 고유 구조.
4. **stale row trap**: 캐처 staleness 재연결 직후나 피드 stall 구간엔 같은 호가가 1초마다
   반복 적재될 수 있다(top-of-book 안 변함). 이런 dead 구간은 임밸런스 변동이 0이라 IC를 0쪽으로
   희석. → 1차 분석은 그대로 두되(보수적), **mid 또는 임밸런스가 직전과 100% 동일한 연속행은
   진단으로 카운트**해서 dead 비율을 보고(너무 높으면 결과 해석에 주의).
5. **bybit tickers delta trap**: bybit는 `tickers.*` delta 채널이라 top-of-book가 안 변해도
   staleness/샘플이 갱신될 수 있음(PROGRESS.md 비차단 한계). bybit 결과는 이 한계 조건부로 읽는다.
6. **다중비교**: 6피드 × 3 horizon = **18개 검정**. 우연히 몇 개는 |IC|가 커 보인다. → **본페로니
   수준**(α=0.05 → 0.05/18 ≈ 0.0028)을 판정 임계로 쓴다. 개별 p<0.05는 "발견"이 아니다.
7. **데이터 충분성 우선**: 비중복 표본수가 적으면 IC는 노이즈다. → 판정 전 **표본수 floor**(아래 5).

## 5. 판정 기준 (사전등록 — 숫자 보기 전 확정, 셋 다 정직)

피드(거래소×심볼)×horizon 단위로 본다. **비중복 표본수 floor = 2,000**
(10s 기준 약 5.5시간치 연속 데이터, 60s 기준 약 33시간치 — 1초 샘플 6피드 몇 주면 충분히 넘김).

각 (피드, horizon)에 대해:
- **IC** = forward return과 임밸런스의 **Spearman 순위상관**(비선형·이상치 강건. Pearson 아님).
- **분위수 스프레드** = 임밸런스 최상위 분위(Q5) 평균 forward return − 최하위 분위(Q1) 평균.
- **단조성** = Q1→Q5 평균 forward return이 단조 증가하는가(가설 부호와 일치).

**3진 판정**:
- **신호 있음 (signal)**: 표본수 ≥ floor AND IC 부호 = 가설(+) AND |IC|의 p < 0.0028(본페로니)
  AND 분위수 스프레드 부호 = 가설 AND Q1→Q5 단조(또는 최소 Q5>Q4>...>Q1의 약단조).
  **그것도 18검정 중 일관된 패턴**(한두 피드만 우연히 뜬 게 아니라 여러 피드에서 같은 부호·유의).
  → **전략 구현 단계로**(아래 8). ⚠️단 IC가 작으면(예: |IC|<0.02) 통계유의해도 **수수료 못 넘을
  가능성** 명시 — 다음 단계에서 taker 0.04%×왕복 = 0.08%를 신호가 포착하는 수익이 넘는지부터 본다.
- **신호 없음 (no-signal)**: 표본수 충분한데 IC가 본페로니 유의 미달 OR 부호 위반 OR 비단조
  → 깨끗한 falsification → **STOP**. "전략으로 짜면 다를지도"로 진행 금지.
- **데이터 부족 (inconclusive)**: 표본수 < floor → 아직 판정 못 함. **며칠 더 적재** 후 재실행.
  (no-signal과 혼동 금지 — 증거 없음 ≠ 증거가 불리.)

⚠️**IC 크기 현실감**: 마이크로구조 신호의 IC는 보통 작다(0.01~0.05면 강한 축). 통계유의(많은 표본)와
경제유의(수수료 후 남는가)는 다르다. 이 분석은 **통계유의(예측력 존재 여부)** 까지만 판정하고,
경제유의(실거래 수익)는 다음 단계(전략+OOS+수수료)에서 본다.

## 6. 방법 (사전등록 — 순수 SQL)

`order_books`가 단순 top-of-book(best_bid/ask + size)라 SQL window 함수로 완결된다.
거래소별 의존성 0, Hetzner의 capture-postgres에서 `psql`로 직접 실행. 산출물:

- **`deploy/analysis/imbalance_ic.sql`**: (거래소,심볼,horizon)별 Spearman IC + 비중복 표본수 +
  본페로니 유의 여부. forward 매칭은 lateral/range 조인(±2초 tol), 비중복은 horizon 간격 추출.
- **`deploy/analysis/imbalance_quantiles.sql`**: 임밸런스 5분위(quintile)별 평균 forward return +
  단조성 + Q5−Q1 스프레드. (거래소,심볼,horizon)별.
- **`deploy/analysis/README.md`**: 실행법 + **데이터 충분성 게이트 쿼리**(표본수 floor 도달 여부를
  먼저 확인 → 도달 전엔 판정 금지).

⚠️**판정은 비중복 IC·분위수로만**(함정 #2). 전체 행 통계는 참고용.

## 7. 다중비교·재현 (판정문에 명시)

- 18 검정 본페로니(α/18). 일관성(여러 피드 같은 부호·유의) 요구.
- **부호 위반(IC가 유의하게 음)**: "역신호 발견"으로 곧장 쓰지 말 것. 가설(1)은 I↑→r↑로 못박았다.
  음의 IC가 나오면 별도 가설로 **한 번도 안 쓴 새 데이터 구간**(다른 기간/심볼)에서 재확인해야
  family-wise 정직성 유지. (사후적 부호 뒤집기 = 거짓양성 양산.)
- **marginal signal**(몇 피드만 약하게): 데이터 더 쌓아 표본수 늘린 뒤 재판정. 저표본 양전은 노이즈
  (가격방향 함정 #4의 교훈 — 저거래/저표본 "양전"은 표본 늘면 사라짐).

## 8. 신호가 살아남으면 — 다음 단계 (이 문서 범위 밖, 미리 못박음)

신호 연구가 PASS여도 **곧장 라이브 아님**. 순서:
1. 신호를 전략으로 구현(`Strategy::evaluate`가 캔들만 받으므로 **오더북 신호용 trait 확장 필요** —
   이건 엔진/머니패스 변경이라 RED→GREEN + 적대적 리뷰 대상).
2. 가격방향 패밀리와 **동일한** 워크포워드 + OOS + 수수료(1×/2×/3×) + 3진판정 + 적대적 리뷰.
3. 거기서도 살아남고 marginal이면 **한 번도 안 쓴 새 홀드아웃** 후에야 testnet.
4. ⚠️**같은 family-wise 규율**: 임밸런스도 변형을 무한정 돌리면 거짓양성. 신호 연구가 죽으면 STOP.

## 9. 한계 (정직)

- top-of-book 1레벨만(전체 depth 아님) — L1 임밸런스는 L2~L10보다 약한 신호일 수 있음. 캐처가
  L1만 적재하므로 이 분석은 L1 조건부.
- 1초 샘플 — 초내(sub-second) 마이크로구조는 못 봄. HFT 영역의 임밸런스 알파는 이 해상도로 안 보임
  (애초에 그건 우리 실행속도로 못 먹음 — 1초가 현실적 하한).
- bybit delta 채널 한계(함정 #5).
- 단일 적재기간(2026-06~) = 단일 레짐. 신호가 살아도 레짐 일반화는 별도 검증.
- 이 분석은 **예측력 존재 여부**만 판정. 실거래 수익(수수료/슬리피지/체결지연 후)은 8단계에서.
